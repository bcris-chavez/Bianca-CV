Acerca de criptografía

La criptografía y el Bitcoin

Bitcoin utiliza elliptic curve cryptography (ECC) y Secure Hash Algorithm 256 (SHA-256) para generar llaves públicas de sus respectivas llaves privadas.

Se utilizan las llaves públicas para crear la dirección del monedero criptográfico que recibirá transacciones de Bitcoins. Las llaves privadas se utilizan para firmar la transacción y asegura que esa dirección monedero le pertenece y por tanto puede hacer uso de los Bitcoins. Estas llaves privadas se almacenan en el monedero criptográfico, el cual permite el acceso a los fondos criptográficos.

Elliptic curve cryptography (ECC)

ECC utiliza una curva matemática especial que es horizontalmente simétrica. Si se dibuja una línea a través de esa curva, se cruzará con la forma hasta en tres ocasiones.

La clave privada de Bitcoin es generada automaticamente, al configurar un nuevo monedero criptográfico, con un número aleatorio de 256 bits. A partir de esta llave se puede generar la llave pública, mediante la multiplicación de curvas elípticas. Para ello se toma un punto de partida en una curva elíptica y se multiplica por la llave privada produciendo así un nuevo punto en la curva. Dicho punto se convierte en la llave pública.

Actualmente no es posible encontrar la llave privada conociendo la llave pública, por lo cual se considera un sistema seguro.

SHA-256

El algoritmo SHA-256 convierte cualquier entrada en un código único de 256 bits de longitud fija, el cual se presenta en formato hexadecimal de 64 caracteres. En el caso de las criptomonedas, las coordenadas de la llave pública ingresan al algoritmo SHA-256.
